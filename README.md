# NETIX.AI — Predictive Maintenance with ML

A machine learning project for predictive maintenance of building systems (HVAC, pumps, AHUs) using both **classification** (fault detection) and **regression** (performance prediction) models.

Datasets are excluded due to GitHub file size limits.

To run the project locally, place datasets in:

data_raw/

## Project Overview

This project trains and evaluates ML models on sensor data from building equipment to:

- **Classification**: Detect fault conditions across HVAC units, pumps, AHUs, and bypass filters
- **Regression**: Predict continuous performance metrics (e.g. PMU LRP outputs)

The project was originally developed in **Google Colab** and has since been migrated to a **local VS Code environment** with relative `pathlib` paths — no Google Drive mounting required.

## Setup Instructions

Follow these steps once before running any notebooks.

### 1. Clone or download the project

If using Git:
```bash
git clone <your-repo-url>
cd NETIX.AI
```

Or simply unzip the project folder and open it in VS Code.

---

### 2. Create a virtual environment

A virtual environment keeps this project's dependencies isolated from your system Python.

**On macOS / Linux:**
```bash
python3 -m venv .venv
```

**On Windows:**
```bash
python -m venv .venv
```

---

### 3. Activate the virtual environment

**On macOS / Linux:**
```bash
source .venv/bin/activate
```

**On Windows (Command Prompt):**
```bash
.venv\Scripts\activate.bat
```

**On Windows (PowerShell):**
```powershell
.venv\Scripts\Activate.ps1
```

You should see `(.venv)` appear at the start of your terminal prompt, confirming activation.

> To deactivate when you're done: simply run `deactivate`

---

### 4. Install dependencies

With your virtual environment active:

```bash
pip install -r requirements.txt
```

This installs all required libraries (pandas, scikit-learn, matplotlib, etc.).

---

### 5. Register the virtual environment as a Jupyter kernel

This makes the `.venv` environment available inside Jupyter notebooks:

```bash
pip install ipykernel
python -m ipykernel install --user --name=netix-venv --display-name "Python (NETIX)"
```

---

### 6. Open the project in VS Code

```bash
code .
```

When you open a notebook (`.ipynb`), click the **kernel selector** in the top-right corner and choose **"Python (NETIX)"** to use the virtual environment you just set up.

## Data

### Raw data (`data_raw/`)

The raw `.xlsx` files are the source of truth. They are **not modified** by any notebook — all processing outputs go to `data_clean/`.

| Subfolder | Equipment | Task |
|-----------|-----------|------|
| `classification/` | AHU 01–03, BP BF 1D/1S, CHW Pump 1–2, OAHU 19-02/37-01 | Fault classification |

## Running the Notebooks

> ⚠️ Always use the notebooks in the **`notebooks/`** folder. The ones in `original/` are legacy Colab versions kept for reference only.

### In VS Code

1. Open `notebooks/NETIX_CSCI323_Classification_main.ipynb` or `notebooks/NETIX_CSCI323_Regression_main.ipynb`
2. Select the **"Python (NETIX)"** kernel (top-right)
3. Click **Run All** (`Ctrl+Shift+P` → "Run All Cells") or run cells one by one

### In Jupyter Lab / Notebook (alternative)

With your virtual environment active:
```bash
jupyter lab
```
Then navigate to `notebooks/` and open the desired notebook.

### Recommended run order

Run classification first, then regression — they are independent but the classification notebook generates some shared utility functions:

1. `NETIX_CSCI323_Classification_main.ipynb`
2. `NETIX_CSCI323_Regression_main.ipynb`

## Notebooks Reference

### `NETIX_CSCI323_Classification_main.ipynb`

Covers fault detection across multiple building system types:

- **Data ingestion**: loads raw `.xlsx` files from `data_raw/classification/`
- **Preprocessing**: cleaning, encoding, train/val/test split
- **Model training**: classifiers (e.g. Random Forest, XGBoost, or similar)
- **Evaluation**: accuracy, F1, confusion matrices
- **Output**: saves processed splits to `data_clean/classification/`

### `NETIX_CSCI323_Regression_main.ipynb`

Covers continuous performance prediction for PMU LRP equipment:

- **Data ingestion**: loads raw `.xlsx` files from `data_raw/regression/`
- **Preprocessing**: feature engineering, normalisation, train/test split
- **Model training**: regression models (e.g. Random Forest Regressor, XGBoost Regressor)
- **Evaluation**: RMSE, MAE, R²
- **Output**: saves processed splits to `data_clean/regression/`

## Troubleshooting

**`ModuleNotFoundError` when running a notebook**
> Make sure you selected the **"Python (NETIX)"** kernel, not your system Python. Re-run `pip install -r requirements.txt` with the venv active.

**Kernel not showing in VS Code**
> Re-run the ipykernel install step:
> ```bash
> python -m ipykernel install --user --name=netix-venv --display-name "Python (NETIX)"
> ```
> Then reload VS Code (`Ctrl+Shift+P` → "Developer: Reload Window").

**`FileNotFoundError` for data files**
> Make sure you are running the notebooks from the **project root** (`NETIX.AI/`), not from inside the `notebooks/` subfolder. The paths are relative to the project root.

**`data_clean/` folder is missing or empty**
> This is expected on first run. The notebooks generate this folder automatically. Just run the notebook from the top.

**Excel read errors on Windows**
> Ensure `openpyxl` is installed:
> ```bash
> pip install openpyxl
> ```
